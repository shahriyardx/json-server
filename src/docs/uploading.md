# Uploading

## How to Upload

1. Sign in with GitHub
2. Navigate to the Upload page
3. Choose **File** mode to drag-and-drop a `.json` file, or **Paste** mode to write/paste JSON content directly
4. Edit the filename if needed
5. Click Upload

On success, the public URL is automatically copied to your clipboard and you're redirected to My JSONs.

## Paste Mode

Switch to **Paste** tab on the upload page to type or paste JSON content. The editor validates JSON in real-time — invalid JSON shows an error and blocks upload. Dragging a file onto the paste mode textarea still loads its content.

## Rules

- **Filename**: letters, numbers, dashes, and underscores only. No spaces.
- **File type**: must be valid `.json`
- **Uniqueness**: filenames are unique per user. Uploading with an existing name overwrites the old file.
- **Storage limit**: 50MB total per account across all files.
- **File limit**: up to 100 files per account.

## Managing Files

Go to **My JSONs** in the dashboard to see all your uploaded files. Each entry shows:

- Filename with public/private status
- File size and creation date
- **Copy URL** — copy the public API URL
- **Download** — download the file as `.json`
- **Docs** — view API documentation for that file
- **Explore** — browse JSON content in a table or tree viewer
- **Versions** — view version history and revert changes
- **Edit** — modify the file content
- **Delete** — remove the file

Use the **Export All ZIP** button to download all files as a single ZIP archive. Use the **search bar** to filter files by name or content.
